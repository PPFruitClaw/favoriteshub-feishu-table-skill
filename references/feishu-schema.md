# 飞书多维表格字段规范

## 子表命名

- `github`
- `x`
- `小红书`
- `抖音`
- `other`

## 字段清单（基础字段）

以下字段应作为所有子表共享的基础字段：

- `标题`：多行文本（主字段，自动生成）
- `所属平台`：单选（`github/x/小红书/抖音/other`）
- `状态`：单选（`已学习/已过期/未学习/重点收藏`，默认 `未学习`）
- `链接`：URL 或文本
- `内容梗概`：多行文本（由 OpenClaw 自主中文概括，不直接复制原文）
- `收藏或星标数量`：整数（各平台若可采集到则填写；未采集到可留空）
- `收录时间`：日期时间

## 平台扩展字段（允许存在）

### `x` 子表

- `作者主页`：URL 字段
- 推荐保存格式：Feishu URL 对象
  - `link=https://x.com/<handle>`
  - `text=<作者原始昵称>`
- 显示文本规则：
  - **严格优先使用作者原始昵称 / 显示名**
  - 不要默认显示成 `@handle`
  - 不要自行润色、翻译、缩写、改写，也不要擅自换成更“好看”的别名
  - 只有在确实拿不到昵称时，才允许退化为 handle
- 迁移规则：
  1. 先批量补 `作者主页`
  2. 做全表缺口检查，确认 `作者主页` 为空的记录数为 `0`
  3. 抽样确认显示与点击正常
  4. 最后才允许删除旧的纯文本 `作者` 列

### `github` 子表

- `项目名称`：文本字段
- 推荐填写：仓库名（repo slug），通常取 GitHub 链接最后一段
  - 例如：
    - `https://github.com/op7418/CodePilot` → `CodePilot`
    - `https://github.com/openclaw/openclaw` → `openclaw`
    - `https://github.com/Martian-Engineering/lossless-claw` → `lossless-claw`
- 注意：
  - `项目名称` 与 `标题` 不是同一字段
  - `标题` 写好了，不代表 `项目名称` 已完成
  - 验收时必须做全表缺口检查，确认 `项目名称` 为空的记录数为 `0`

> 说明：基础字段应统一，但平台扩展字段允许存在；不要再把“所有子表字段必须完全一致”当成硬约束。

## 初始化建议

1. 新建多维表格应用（app）。
2. 在默认表或目标子表中创建上述字段。
3. 为每个平台准备同名子表，字段结构保持一致。
4. 首次写入前用 `所属平台 + 链接` 做去重检查，避免重复记录。

## 写入映射

- `title` -> `标题`
- `platform` -> `所属平台`（`xiaohongshu -> 小红书`，`douyin -> 抖音`）
- `status` -> `状态`（默认 `未学习`）
- `link` -> `链接`
- `summary` -> `内容梗概`
- `favorite_or_star_count` -> `收藏或星标数量`
- `ingested_at` -> `收录时间`
