# 飞书多维表格字段规范

## 子表命名

- `github`
- `x`
- `小红书`
- `抖音`
- `other`

## 字段清单（所有子表一致）

- `标题`：多行文本（主字段，自动生成）
- `所属平台`：单选（`github/x/小红书/抖音/other`）
- `状态`：单选（`已学习/已过期/未学习`，默认 `未学习`）
- `链接`：URL 或文本
- `内容梗概`：多行文本（优先 LLM 中文概括，不直接复制原文）
- `收藏或星标数量`：整数（各平台若可采集到则填写；未采集到可留空）
- `收录时间`：日期时间

## 初始化建议

1. 新建多维表格应用（app）。
2. 在默认表或目标子表中创建上述字段。
3. 为每个平台准备同名子表，字段结构保持一致。
4. 首次写入前用 `所属平台 + 链接` 做去重检查，避免重复记录。

## 写入映射

- `title` -> `标题`
- `platform` -> `所属平台`（`xiaohongshu -> 小红书`，`douyin -> 抖音`）
- `status` -> `状态`（默认 `未学习`）
- `link` -> `链接`
- `summary` -> `内容梗概`
- `favorite_or_star_count` -> `收藏或星标数量`
- `ingested_at` -> `收录时间`
